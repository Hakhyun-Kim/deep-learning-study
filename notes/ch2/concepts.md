# ch2 개념 사전 — 전체 파인튜닝에 필요한 용어

> ch1 개념([../ch1/concepts.md](../ch1/concepts.md))을 안다고 가정하고,
> exp02(전체 파인튜닝)에서 새로 만나는 것만 정리.

## exp01 vs exp02 — 뭐가 달라지나

| | exp01 (fc만) | exp02 (전체) |
|---|---|---|
| 명령 | `--freeze-backbone` (lr 기본 1e-3) | `--lr 1e-4` (얼림 없음) |
| 학습 파라미터 | 20,520개 (0.2%) | 11,197,032개 (100%) |
| 학습률 | 1e-3 | **1e-4 (10배↓)** |
| 위험 | 표현력 부족 (69% 한계) | 사전학습 특징 훼손, 과적합 |
| 결과 | 69.4% | ? (목표: 80% 근접) |

코드는 한 줄도 안 바뀐다 — `build_model()`의 `if freeze_backbone:` 분기를
안 타는 것뿐. **실험 간 차이가 커맨드라인 옵션으로만 남는** 설계의 이점.

## 손실 지형 (loss landscape)

- 파라미터 값에 따라 손실이 어떻게 변하는지를 지형(산과 골짜기)으로 상상한 것.
  학습 = 이 지형에서 낮은 곳 찾아 내려가기.
- 사전학습된 백본은 이미 **좋은 골짜기 근처**에 있다. 그래서 큰 걸음(높은 lr)이면
  골짜기를 지나쳐 튕겨 나가고, 작은 걸음(1e-4)으로 바닥을 더듬어야 한다.
- 무작위 초기화(exp01의 fc층)는 지형 한복판 아무 데서나 출발 → 큰 걸음이 효율적.
- 관련: [q03](../ch1/q03-full-finetune-low-lr.md)

## 파국적 망각 (catastrophic forgetting)

- 신경망이 새 데이터를 학습하면서 **이전에 배운 것을 덮어써 잃어버리는** 현상.
  높은 lr로 전체 파인튜닝하면 ImageNet에서 배운 "보는 능력"이 뭉개진다.
- **테슬라 예시**: FSD를 새 도시(예: 서울) 데이터로만 재학습하면 기존 도시 실력이
  망가질 수 있다. 그래서 재학습 때 **새 데이터 + 기존 데이터를 섞어서**(data mix)
  학습하고, 배포 전 회귀 테스트(기존 상황 시험)를 돌린다. 우리의 "낮은 lr"은
  같은 문제에 대한 더 단순한 안전장치다.

## weight decay — AdamW의 "W"

- 매 갱신마다 가중치를 **살짝 0 쪽으로 당기는** 규제(regularization).
  가중치가 커지는 건 "특정 입력 패턴에 과도하게 확신"하는 것이라, 이를 억눌러
  과적합을 줄인다.
- 이름 그대로 AdamW에 내장되어 있고(기본 0.01), 우리는 이미 쓰고 있었다.
  exp02는 학습 파라미터가 546배로 늘어 과적합 위험이 커지므로 이 규제가
  실질적으로 중요해진다.
- 비유: 시험공부할 때 "한 문제 유형에 올인"하지 못하게 골고루 공부시키는 것.

## 층별 학습률 (discriminative learning rate)

- 층마다 다른 lr을 주는 것: 백본은 아주 작게(1e-5), fc는 크게(1e-3).
  "많이 움직여야 하는 층"과 "조금만 움직여야 하는 층"을 각자 대접.

```python
optimizer = torch.optim.AdamW([
    {"params": backbone_params, "lr": 1e-5},
    {"params": model.fc.parameters(), "lr": 1e-3},
])
```

- exp02는 단일 lr 1e-4로 시작 (단순한 것부터). 성능이 아쉬우면 다음 실험 후보.
- 우리는 이미 **2단계 파인튜닝**을 하고 있는 셈이다: exp01에서 fc를 먼저
  수렴시켰고(1단계), exp02는 거기서 이어가는 대신 처음부터 다시 돌리지만,
  exp01 체크포인트에서 이어 학습하는 변형도 가능하다.

## 학습률 스케줄러 (lr scheduler)

- 학습 도중 lr을 **점점 줄여가는** 장치. 초반엔 성큼성큼, 골짜기 바닥 근처에선
  잰걸음 — 고정 lr보다 마지막 수렴이 좋아지는 경우가 많다.
- 대표: cosine annealing (부드럽게 0으로), step decay (계단식).
- exp02는 고정 lr로 시작. 스케줄러는 ch3 실험 후보 (한 번에 한 변수만!).

## 과적합 감시 — 곡선 4개 읽는 법

전체 파인튜닝은 파라미터가 많아 과적합이 진짜 위협이 된다.
[대시보드](https://hakhyun-kim.github.io/deep-learning-study/)에서 볼 순서:

1. **test_loss가 오르기 시작하는 에폭** — 과적합의 가장 이른 신호.
   test_acc는 한동안 유지될 수 있어서 loss가 먼저 경고한다.
2. train_loss와 test_loss의 **간격이 벌어지는 속도** — 외우기 시작하는 정도.
3. test_acc의 최고점 — best 체크포인트가 자동 저장되므로, 과적합이 와도
   "가장 좋았던 순간"은 안전하다 (`checkpoints/exp02_full_finetune_best.pt`).

- **테슬라 예시**: 학습 데이터(미국 도로)에서만 잘하고 처음 보는 상황에서 못하면
  실전 투입 불가 — 그래서 검증은 반드시 학습에 안 쓴 주행 데이터로 한다.
  test_loss 상승 감시는 우리 규모의 "배포 전 검증"이다.

## GPU 메모리와 배치 크기

- 역전파를 하려면 순전파 때의 **중간 계산값(활성값)을 전부 기억**해야 한다.
  exp01은 fc 앞 기울기가 필요 없어 일부를 버릴 수 있었지만, exp02는 17개
  합성곱층 전부의 활성값 + 기울기 + AdamW 상태(파라미터당 2개)를 들고 있어야 한다.
- RTX 3050 Ti는 4GB — ResNet18 batch 32는 보통 괜찮지만, 만약
  `CUDA out of memory`가 나면 `--batch-size 16`으로 낮추면 된다
  (배치를 줄이면 lr도 같이 낮추는 게 정석이지만, 2배 차이 정도는 그대로 둬도 무방).

## 통제 실험 — 한 번에 한 변수만

- exp02(lr 1e-4)와 exp02b(lr 1e-3)는 **lr만 다르고 전부 동일**해야 한다.
  두 개 이상 바꾸면 결과 차이가 뭐 때문인지 알 수 없다.
- 우리 대시보드가 실험별 config를 저장하는 이유이자, 실험 이름을
  `exp02b_full_lr1e-3`처럼 짓는 이유.
- **테슬라 예시**: 새 모델 배포 전 A/B로 이전 버전과 같은 조건에서 비교하는
  것과 같은 원리.
