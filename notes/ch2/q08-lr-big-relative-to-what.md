# Q8. lr 1e-3이 "큰 발자국"이라는 건 어떻게 아나? 크다/작다는 상대적인 값 아닌가?

> 2026-07-11 · 관련: exp02 vs exp02b, [q03](../ch1/q03-full-finetune-low-lr.md), [`scripts/measure_backbone_drift.py`](../../scripts/measure_backbone_drift.py)
> 상태: ✅ 정리됨 — 가중치 이동량을 직접 측정해서 확인

## 질문

exp02b(lr 1e-3)의 붕괴를 catastrophic forgetting으로 설명했는데, 애초에
"1e-3 = 큰 걸음"이라는 걸 어떻게 알 수 있나? 0.001이면 절대값으로는 아주 작은
수인데. 값이 큰지 작은지는 상대적인 것 아닌가?

## 한 줄 답

맞다, 완전히 상대적이다. 학습률은 **두 가지 기준에 대한 비율**로만 의미가 있다 —
① 가중치 자체의 크기, ② 목적지(손실 골짜기 바닥)까지 남은 거리. 그리고 "컸는지"는
추측이 아니라 **측정**할 수 있다.

## ① 가중치 크기와 비교 — 0.001은 사실 작지 않다

사전학습된 ResNet18의 합성곱 가중치는 놀랄 만큼 작은 수들이다
(`scripts/measure_backbone_drift.py` 출력):

```
layer3.0.conv1 가중치: 표준편차 0.029, 평균 크기 0.021
```

AdamW는 기울기 크기를 정규화해서 파라미터 하나를 스텝당 **대략 lr만큼** 움직인다.
그러면 곱셈 두 번으로 감이 온다:

- 한 스텝에 1e-3 이동 = 평균 가중치(0.021)의 **약 5%** — 한 걸음에 5%면 산책이 아니다
- 우리 학습은 에폭당 125스텝 × 10에폭 = **1,250스텝**. 스텝마다 방향이 조금씩
  달라도, 1e-3 × 1250 = 1.25라는 "이동 예산"은 가중치 크기 0.021의 **60배**다
- lr 1e-4면 예산이 0.125 — 여전히 크지만 한 자릿수 줄어서 "조정" 범위에 들어온다

즉 lr의 크고 작음은 **가중치의 세계가 얼마나 미세한 스케일인지**와 비교해야 보인다.
0.001은 우리 일상 감각으로는 작지만, 0.02짜리 가중치의 세계에서는 크다.

## ② 실측 — 백본이 실제로 얼마나 이동했나

"큰 발자국"이 비유가 아니라는 걸 확인하려고, 두 실험의 체크포인트를 ImageNet
가중치와 직접 비교했다 (상대 L2 거리, `python scripts/measure_backbone_drift.py`):

| | exp02 (lr 1e-4) | exp02b (lr 1e-3) |
|---|---|---|
| 백본 전체 이동량 | **16.8%** | **97.6%** |
| 첫 합성곱 (conv1) | 1.2% | 9.7% |
| 마지막 합성곱 (layer4.1.conv2) | 39.8% | 177.4% |

- exp02b의 97.6%는 "ImageNet에서 출발한 지점과, 그냥 0에서 출발한 지점의 중간쯤까지
  가버렸다"는 뜻 — 사전학습 가중치가 사실상 **다시 쓰였다**. 이게 forgetting의 실체다.
- exp02의 16.8%는 원래 자리 근처에 머물렀다 — "fine"-tuning이라는 이름값.
- 보너스 발견: 층마다 이동량이 다르다. 첫 층(선·질감 같은 범용 특징)은 거의 안
  움직이고(1.2%), 마지막 층(과제 특화 특징)은 많이 움직인다(39.8%). ch2 concepts의
  **층별 학습률**(discriminative lr) 아이디어가 왜 나왔는지 곡선 없이도 보인다.

## ③ 그럼 돌리기 전에는 어떻게 아나? — 사실 아무도 모른다

솔직한 답: 실무에서도 lr이 "큰지"는 **돌려보고 곡선으로 진단**한다.
lr이 하이퍼파라미터 튜닝 1순위인 이유다. 신호는 이미 배웠다:

- 손실이 **출렁이면** (exp02b의 test_loss 1.83 → 2.06 → 1.69) 걸음이 골짜기
  폭보다 크다는 뜻 — 바닥을 사이에 두고 왔다 갔다
- 손실이 **발산하면** 아예 골짜기 밖으로 튕겨 나간 것
- 너무 작으면 곡선이 기어간다 — 이건 시간만 손해라 덜 위험

다만 사전지식은 있다: Adam 계열의 기본값 1e-3은 **무작위 초기화에서 밑바닥부터**
학습하는 상황에 맞춰진 값이다. 출발점이 이미 좋은 파인튜닝은 10~100배 낮추는 게
표준 관행 — exp01→exp02에서 우리가 한 그대로.

## 테슬라로 연결

핸들을 30도 꺾는 게 "큰 조작"인가? — 주차장에서 크게 벗어나 있을 때는 정상 조작이고,
고속도로에서 이미 차선 중앙에 있을 때는 사고다. **조작의 크기는 절대각이 아니라
"목표까지 남은 오차" 대비 비율**로 정해진다. exp01의 fc층(무작위 출발 = 크게 벗어남)에게
1e-3은 주차장의 30도였고, exp02b의 백본(이미 차선 중앙)에게 같은 1e-3은 고속도로의
30도였다.

## 스스로 점검 (복습용)

- [ ] "lr이 크다"를 판단하는 두 가지 비교 기준을 말할 수 있다
- [ ] 사전학습 가중치의 전형적 크기(~0.02)와 lr 1e-3의 관계를 계산으로 보일 수 있다
- [ ] 이동량 16.8% vs 97.6%가 각각 뭘 의미하는지 설명할 수 있다
- [ ] 학습 곡선에서 "lr이 너무 크다"는 신호 2개를 안다
- [ ] 같은 1e-3이 exp01에서는 괜찮고 exp02b에서는 파괴적이었던 이유를 말할 수 있다
