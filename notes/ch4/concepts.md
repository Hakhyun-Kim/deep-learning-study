# ch4 개념 사전 — 마무리에 필요한 용어

> ch3까지의 개념을 안다고 가정. ch4에서 새로 등장한 것만.

## 앙상블과 상보성 (ensemble, complementarity)

- **앙상블(ensemble)** = 여러 모델(또는 여러 시야)의 예측을 합쳐 하나의 답을
  내는 것. 우리 최종 시스템: `0.3 × 사람시야 + 0.7 × 전체시야(TTA)`.
- 통하는 조건은 "각자가 잘하는 것"이 아니라 **오답이 서로 겹치지 않는 것**
  (error decorrelation). 고전 정리: Dietterich, *"Ensemble Methods in Machine
  Learning"* (2000) — 개별 모델이 무작위보다 낫고 **서로 다르게 틀리면**,
  합친 것은 개별보다 낫다.
- **우리 실측이 교과서 사례**: 사람 시야 단독 70.79%(전체 시야보다 5%p 낮음)를
  반반 섞었더니 79.37%, TTA와 7:3으로 섞으니 **80.71%**. 약한 모델도 **다른
  관점**이면 재료다 — 단독 점수만 보고 버리지 말 것.
- **테슬라 예시**: 카메라 8대 + (과거) 레이더의 센서 퓨전이 같은 원리 —
  각 센서가 다른 조건(역광/거리)에서 다르게 틀리니, 합치면 각각보다 강해진다.

## 검증 셋 (validation set) — "test는 한 번만"

- 우리가 앙상블 가중치 w를 **test set 성적을 보고** 골랐다(w=0.3). 이 순간
  test set은 더 이상 "처음 보는 시험"이 아니다 — w라는 결정이 test에 맞춰졌으니
  점수가 살짝 낙관적으로 오염된다 (**test set leakage / 간접 과적합**).
- 올바른 절차: train으로 학습 → **validation split**(train에서 떼어둔 부분)으로
  w·하이퍼파라미터 선택 → test는 **맨 마지막에 딱 한 번**.
- 우리의 정직한 처리: w=0.2~0.4 구간 전체가 80%를 넘어 결론("80% 돌파")은
  강건하지만, "80.71%"라는 소수점은 ±로 읽어야 한다고 기록해둠.
- **테슬라 예시**: 새 빌드를 검증 주행 데이터로 튜닝하고, 최종 판정은 별도의
  홀드아웃 시나리오(본 적 없는 도시·상황)로만 내리는 것과 같다.

## 추론 비용 (inference cost)

- 최종 시스템은 이미지 1장에 **순전파 6회**(전체 시야 5-crop + 사람 시야 1회).
  정확도 +4.9%p의 값이 계산량 6배라는 뜻 — 공짜가 아니다.
- 배포 상황에 따라 트레이드오프가 달라진다: 사진 앱 배치 태깅이면 OK,
  실시간(테슬라 프레임당 수십 ms 예산)이면 불가 — 그쪽은 증류(distillation)나
  단일 패스 구조로 같은 지식을 압축하는 게 다음 단계가 된다.

## 2단계 파이프라인 (two-stage pipeline: detect → classify)

- 우리 최종 시스템의 숨은 가정: **사람 bbox가 주어진다** (공식 주석 사용).
  실전이면 사람 탐지기(person detector)가 앞단에 필요하다 — 탐지 실패는
  그대로 분류 실패로 이어지는 새 오류 원천.
- 이는 속임수가 아니라 산업 표준 구조다: 테슬라도 탐지(차량·보행자) →
  잘라낸 영역의 세부 분류(브레이크등·방향지시등) 순서로 돈다. 다만 기록엔
  "우리 80.71%는 탐지기 성능을 포함하지 않은 숫자"라고 남겨야 정직하다.
- Stanford 40의 원 과제 정의 자체가 bbox 제공을 전제한다 (Yao et al., ICCV 2011).

## 모델 카드 (Model Cards) — 한계를 문서화하는 관행

- Mitchell et al., *"Model Cards for Model Reporting"* (FAT\* 2019): 모델을
  공개할 때 성능 숫자만이 아니라 **어떤 데이터로, 어떤 조건에서, 어디까지
  믿을 수 있고 어디서 실패하는지**를 표준 양식으로 남기자는 제안. 현재
  HuggingFace 등에서 사실상 표준.
