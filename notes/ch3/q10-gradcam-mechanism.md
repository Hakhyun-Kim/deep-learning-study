# q10 — Grad-CAM은 "모델이 본 이미지 부분"을 어떻게 찾아내나?

관련: [ch3 concepts — Grad-CAM](concepts.md#grad-cam-열지도로-근거-보기),
[q09 — 배경 지름길](q09-shortcut-learning.md) · [`scripts/gradcam.py`](../../scripts/gradcam.py)

## 무슨 질문인가

exp02 진단에서 Grad-CAM이 "phoning_253은 전화기가 아니라 **뒤 벽의 글씨**를
보고 있었다"를 잡아냈다([q09](q09-shortcut-learning.md)). 그런데 Grad-CAM은
대체 **어떻게** 이미지의 그 부분을 찾은 걸까? 모델이 "여기 봤어요"라고 말해주는
것도 아닌데.

## 한 문장 직관

Grad-CAM은 없는 정보를 만들지 않는다. 모델이 **이미 계산해 둔 마지막 특징 지도**를
꺼내서, 그중 **이번 예측에 도움이 된 부분만** 골라 밝게 칠한다.
재료는 딱 둘: **① 특징 지도(무엇이 어디에)** × **② 기울기(그게 이 예측에 얼마나 중요했나)**.

## 재료 ① — 특징 지도 (activation): "무엇이 어디에"

ResNet18은 224×224를 층층이 압축해, 마지막 블록 `layer4`에서 **7×7 지도 512장**을
내놓는다 ([gradcam.py:67](../../scripts/gradcam.py) `A = (512, 7, 7)`).

- **7×7** = 원본을 32배 줄인 거친 격자. 칸 하나가 원본 약 32×32px 영역에 대응
  (왼위·가운데·오른아래…). → **"어디"** 정보가 여기 남아 있다.
- **512장** = 서로 다른 것에 반응하는 탐지기들. 어떤 채널은 얼굴, 어떤 채널은
  글씨, 어떤 채널은 접시 모양에 켜진다. → **"무엇"** 정보.

즉 특징 지도만으로 "어디에 무엇이 있나"는 이미 들어 있다. 모르는 건 512장 중
**이번 예측에 어느 게 중요했나**뿐.

## 재료 ② — 기울기 (gradient): "그게 얼마나 중요했나"

핵심은 미분이다. [gradcam.py:63](../../scripts/gradcam.py):

```python
logits[0, class_idx].backward()   # "이 클래스 점수"를 기준으로 역전파
```

`backward()`는 **"이 채널 값이 조금 커지면 예측 점수가 얼마나 오르나?"**를
512개 채널 전부에 대해 구해준다 = 기울기.

- 점수를 크게 끌어올리는 채널 → 기울기 큼 → **중요한 근거**
- 점수와 상관없는 채널 → 기울기 ≈ 0 → **무관**

[gradcam.py:68](../../scripts/gradcam.py)에서 채널마다 7×7 위치를 평균 내
**채널 하나당 중요도 숫자 하나**로 만든다:

```python
weights = grads["v"][0].mean(dim=(1, 2))   # 채널 512개의 중요도
```

## 합치기 — 가중 합

[gradcam.py:69](../../scripts/gradcam.py):

```python
cam = F.relu((weights[:, None, None] * A).sum(0))
```

말로 풀면: **512장의 특징 지도를 각자의 중요도로 곱해 겹쳐 쌓는다.**
중요한 채널(예: 얼굴 탐지기)은 크게, 무관한 채널은 거의 0으로 곱해져 사라진다.
다 더하면 7×7 한 장이 남고 **근거가 있던 칸이 밝게** 뜬다.

마지막으로 [gradcam.py:71](../../scripts/gradcam.py)에서 거친 7×7을 224×224로
부드럽게 늘려(`interpolate`) 원본에 겹치면, 우리가 보는 빨강-파랑 열지도가 된다.
**빨강 = 근거가 있던 곳, 파랑 = 예측과 무관.**

## 스스로 생각해보기

- 왜 하필 **마지막 층(`layer4`)**을 쓸까? 더 앞의 층(56×56로 해상도가 더 높은)을
  쓰면 열지도가 더 정밀해질 텐데, 왜 안 그럴까?
- [gradcam.py:69](../../scripts/gradcam.py)의 `F.relu`를 **빼면** 어떤 그림이
  나올까? 무엇을 못 보게 될까?
- 같은 이미지로 `compute_cam(..., pred)`와 `compute_cam(..., true)`를 따로 그리면
  ([gradcam.py:121-122](../../scripts/gradcam.py)) 왜 열지도가 달라질까?

### 왜 마지막 층(`layer4`)인가 — 해상도 vs 의미의 절충

- 앞쪽 층(layer1, 56×56)은 해상도는 높지만 **선·모서리 같은 저수준 무늬**에만
  반응한다. "행동"의 근거로는 의미가 없다 — 근거지가 아니라 질감 지도가 나온다.
- 마지막 층은 7×7로 거칠지만 채널이 **"얼굴", "글씨판", "접시" 같은 의미 있는
  개념**에 반응하고, 그러면서 **위치 정보가 아직 대충 남아** 있다(전역 풀링 직전).
- 그래서 "무엇을 보고 판단했나"의 근거지로 마지막 합성곱 블록이 딱 맞는다.
  거친 7×7은 [gradcam.py:71](../../scripts/gradcam.py)에서 부드럽게 키워 가린다.

### `F.relu`를 빼면

relu는 **음수(=점수를 깎는 근거)를 잘라내** "점수를 올린 곳"만 남긴다. 빼면
"이 클래스를 **반대**로 미는 영역"까지 섞여, 열지도가 번지고 근거가 흐려진다.
우리가 보고 싶은 건 "왜 이 클래스라고 **찍었나**"이므로 양의 기여만 본다.

### pred vs true 열지도가 다른 이유

기울기의 기준([gradcam.py:63](../../scripts/gradcam.py)의 `class_idx`)이 다르기
때문. "writing_on_a_board 점수를 올린 채널"과 "phoning 점수를 올린 채널"은 서로
다르니, 켜지는 위치도 다르다. **phoning_253에서 pred(writing) 열지도는 뒤 글씨에,
true(phoning) 열지도는 손·전화기에** 뜬다 — 이 둘을 나란히 놓는 게
[q09](q09-shortcut-learning.md)의 지름길 진단을 만든 핵심 장치다.

### 테슬라로 옮기면

FSD가 갑자기 브레이크를 밟았을 때:
- **특징 지도** = 인식망이 프레임에서 잡은 것들(앞차·그림자·표지판)이 화면 어디에.
- **기울기** = 그중 "브레이크" 결정을 실제로 밀어붙인 게 무엇인가.
- **Grad-CAM** = 둘을 곱해 "아 저 **도로 위 그림자**를 장애물로 착각했구나"를 히트맵으로.

개입률(숫자)만 보면 원인을 못 고친다. **무엇을 보고 반응했나**를 봐야 고친다 —
우리가 exp02에서 한 진단의 대형 버전이다.

### 한 줄 요약

> Grad-CAM = (마지막 특징 지도 = 무엇이 어디에) × (기울기 = 이 예측에 얼마나 중요) →
> 겹쳐 더하면 "근거가 있던 위치"가 밝아진다. 새로 찾는 게 아니라 **이미 있던
> 특징을 예측 기준으로 걸러내는** 것.
