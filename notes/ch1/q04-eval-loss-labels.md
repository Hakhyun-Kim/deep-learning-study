# Q4. 평가 때 손실은 어떻게 계산하지? 정답이 없는데?

> 2026-07-10 · 관련: [`src/train.py`](../../src/train.py)의 `evaluate()`

## 한 줄 답

test set에도 **정답(라벨)이 있다**. "학습에 안 쓴다"와 "정답이 없다"는 다른 말 —
정답을 **채점에만 쓰고 가중치 갱신에는 안 쓰는** 것이 평가다.
정답이 진짜로 없는 상황은 평가가 아니라 **추론(배포)**이고, 그때는 손실을
계산할 수 없다(할 필요도 없다).

## 세 가지 상황 구분

| | 학습 (train) | 평가 (evaluate) | 추론 (inference/배포) |
|---|---|---|---|
| 정답 라벨 | 있음 | **있음** | 없음 |
| 손실 계산 | O | **O (채점용)** | X — 계산 불가 |
| 역전파·가중치 갱신 | O | X | X |
| 예시 | train 4,000장 | test 5,532장 | 새 사진 1장을 모델에 넣어봄 |

## 코드로 확인

test set도 train과 똑같이 split 파일에서 (이미지, 라벨) 쌍으로 만들어진다.
[`src/dataset.py`](../../src/dataset.py)의 `Stanford40("test")`가 읽는
`data/ImageSplits/<클래스명>_test.txt`가 test의 **정답지**다.

그래서 [`evaluate()`](../../src/train.py)도 손실을 train과 같은 식으로 계산한다:

```python
@torch.no_grad()                      # 기울기 계산 안 함
def evaluate(model, loader, criterion, device):
    model.eval()
    for images, labels in loader:     # ← test 로더도 labels를 준다
        outputs = model(images)
        loss = criterion(outputs, labels)   # 손실 계산은 동일
        # loss.backward() 없음, optimizer.step() 없음 ← 이게 학습과의 차이
```

라벨이 하는 일이 다르다: 학습에서는 라벨이 역전파를 통해 **가중치를 바꾸지만**,
평가에서는 점수 계산에만 쓰이고 모델에는 아무 영향을 못 준다.
`@torch.no_grad()`가 그걸 구조적으로 보장한다.

## 그럼 진짜 정답이 없으면?

배포된 모델에 새 사진을 넣는 상황(추론)에서는:

- 손실·정확도는 **정의 자체가 안 된다** (비교할 정답이 없으니)
- 모델은 40개 점수만 내놓고, 우리는 `argmax`로 가장 높은 클래스를 답으로 쓴다
- 대신 softmax 확률로 "얼마나 확신하는지"는 볼 수 있다
  (예: 낚시 92% vs 낚시 31% — 후자는 못 믿을 예측)

정리하면: **손실은 "정답을 아는 사람이 매기는 점수"**다. 학습·평가는 정답지를
가진 우리가 채점하는 단계고, 추론은 정답지 없이 실전에 내보내는 단계.
test set은 "추론 상황을 미리 흉내 내되, 채점은 가능하게" 만든 장치다 —
모델 입장에서는 처음 보는 이미지지만, 우리 손에는 정답지가 있다.
