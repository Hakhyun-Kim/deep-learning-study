# q11 — 왜 요즘 LLM은 3개월 단위로 이렇게 빠르게 좋아지나? "질답 데이터가 쌓여서"인가?

관련: [q10 학습 패러다임](q10-learning-paradigms-and-llms.md) ·
[q06 테슬라 데이터 엔진](q06-tesla-rule-to-e2e.md)

> q10에서 "Claude = 사전학습(자기지도) → SFT(지도) → RLHF(강화)"까지 정리했다.
> 이 질문은 그 3단계 **자체**가 아니라, **최근 체감되는 급성장은 어디서 오나**를 다룬다.

## 짧은 답: "대화 데이터 축적"이 주 원인은 아니다

일반 사용자와의 대화 원본을 그대로 모아 다음 모델을 학습시키는 게 핵심 동력은
**아니다.** 회사마다 다르지만 보통 원본 대화를 검증 없이 바로 쓰지 않고,
별도로 정제된 선호 데이터([q10의 RLHF 3단계](q10-learning-paradigms-and-llms.md))를
쓴다. 진짜 동력은 최근 새로 생긴 축들이다.

## 최근 급성장의 4가지 진짜 축

| 축 | 무엇이 달라졌나 | 왜 빠른가 |
|---|---|---|
| **① 추론 시점 연산 (test-time compute)** | 답을 즉시 뱉지 않고 **먼저 길게 "생각"**하고 답함 (extended thinking 등) | 재학습 없이 "더 오래 생각하게" 하는 것만으로 성능이 뜀 — q10의 3단계와 별개 축 |
| **② 합성 데이터 / 자가검증 루프** | 모델이 수학·코드 문제를 스스로 풀고 **정답을 실행/검증**해서 맞은 것만 학습 데이터로 재사용 | 사람이 라벨 달 필요 없이 **모델이 모델을 가르침** — 알파고 자가대국과 같은 구조 |
| **③ 데이터 양보다 질** | 인터넷 전체보다 **엄선·정제된 데이터**가 더 잘 통한다는 게 알려짐 | 같은 컴퓨팅으로 더 똑똑해짐 (원료 개선) |
| **④ 랩들 간 경쟁** | Anthropic·OpenAI·Google 등이 매달 새 버전을 냄 | 순수 기술보다 **출시 속도 자체**가 체감 변화를 키움 |

이 중 **①번(추론 시점 연산)이 최근 가장 큰 반전이다.** 예전엔 "모델을 더 크게,
더 많이 학습시켜야" 똑똑해졌는데, 이제는 **학습은 그대로 두고 "답하기 전에 얼마나
오래 생각하게 허용하느냐"**만 조절해도 성능이 크게 뜬다. 이건 q10의 사전학습/
파인튜닝과는 완전히 다른 축이라, "대화 데이터가 쌓여서"로는 설명이 안 된다.

## 테슬라로 연결하면

테슬라가 차량 함대에서 개입(disengagement) 데이터를 모아 재학습하는 것 —
이게 사용자 상호작용에서 배우는 것과 제일 비슷한 그림이다. 하지만 그것도
**원본을 그대로 쓰는 게 아니라, 엣지케이스를 골라내고 라벨링해서** 쓴다.
LLM의 RLHF도 마찬가지로, 실사용 대화 자체보다는 **거기서 뽑은 "어려운 케이스"를
사람이 다시 평가**하는 정제 과정이 핵심이다.

## q10의 틀로 다시 보면 — 정답 신호의 원천이 하나 늘었다

q10에서 "정답 신호가 어디서 오나"로 패러다임을 나눴다. 최근 급성장은 이렇게 요약된다:

> 예전엔 "사람이 채점"뿐이었는데, 이제 **"실행해보고 검증"**(코드가 돌아가나,
> 수학 답이 맞나)이라는 **자동 채점 신호**가 추가돼서, 사람 개입 없이도
> 모델이 스스로를 훈련시킬 수 있게 됐다. 이게 ②(합성 데이터/자가검증 루프)의 정체다.

## 한 줄 복습

- "대화 데이터가 쌓여서"는 절반만 맞다 — 원본이 아니라 정제된 선호 데이터가 쓰인다.
- 진짜 축: 추론 시점 연산(①), 자가검증 합성 데이터(②), 데이터 질(③), 경쟁 속도(④).
- ①·②는 q10의 3단계(사전학습/SFT/RLHF)와 **별개의 새로운 축**이라는 게 핵심.
